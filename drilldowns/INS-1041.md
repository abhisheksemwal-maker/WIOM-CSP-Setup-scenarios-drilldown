## Why exit is inline (not overflow) for this state only
User direction: this is the state where a CSP who realises the task isn't viable needs the fastest escape. Putting exit under a triple-dot menu on the empty state adds one tap of friction that isn't warranted when there's no work done yet. Every other state keeps exit in the overflow menu (two-tap intent gate).
