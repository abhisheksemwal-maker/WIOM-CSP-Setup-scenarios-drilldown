## Slot section
Shows the **new submitted slots** after the CSP re-submits. Bug caught and fixed this session: previously the section still showed old `EXPIRED` siblings after resubmit because the `onSlotSubmitted` callback was dropping slot data through `{ _, _, _, _ -> onSlotSubmitted(task.id) }`. Fixed by threading slot data through:

```kotlin
onSlotSubmitted = { id, s1d, s1t, s2d, s2t ->
    homeViewModel.onSlotSubmitted(id, s1d, s1t, s2d, s2t)
}
```

Across 4 files: `AppNavGraph`, `TaskDrilldownScreen`, `HomeScreen` (`ActionSheetRouter`), `HomeViewModel`.
