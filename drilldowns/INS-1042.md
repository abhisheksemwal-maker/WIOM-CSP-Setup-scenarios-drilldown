# INS-1042 — AWAITING_CUSTOMER_SELECTION

## Scenario
CSP has submitted two slot proposals to the connection. Nothing more for the CSP to do until the customer picks one. Previously this state had a `जवाब बाकी` chip that looked too active; redesign tones it down.

## Deadline pill
`1-2 दिन बाकी` (or task-specific TAT).

## Slot section
Both proposed slots shown with grey "pending" text styling (not purple chip).

Both slots display secondary status line `पेंडिंग है` per user direction: "since customer hasn't chosen any state, the pending hai state makes more sense for both times. the style of pending hai should be consistent, grey text suits better instead of purple chip design."
