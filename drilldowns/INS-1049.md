# INS-1049 — SCHEDULING_FAILED (hidden from CSP feed)

## Scenario
A system-internal state reached when the connection never confirms any slot and Wiom gives up on the auto-schedule retry chain. CSP has **no action to take** in this state.

## Design decision: hide from CSP feed
User direction: "INS 1049, instead of Wiom dubara koshish karega, can we use connection ne slot confirm nahi kiya. Additionally, we can hide this task and drill down from CSP".

Implementation:
```kotlin
tasks.filter { it.currentState != "SCHEDULING_FAILED" }
```
in `MockTaskRepository.buildTaskFeed()`.

The state is **still visible in the mock repo for testing** and the drilldown renders correctly if you deep-link to it — but the home feed never surfaces it in the redesign flavor.

## CTAs
None. State is informational-only.

## Why filter rather than delete
Preserving the state in the enum + mock repo lets QA and future AI sessions test deep-link behaviour and confirm the filter still works. Hiding via the feed filter is a one-line change that's easy to reverse if product decides to surface it later.
