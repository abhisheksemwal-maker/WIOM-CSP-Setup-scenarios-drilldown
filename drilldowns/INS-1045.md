# INS-1045 — DELEGATED_OVERDUE

## Scenario
CSP delegated the install to a team member (e.g. Rajesh). The executor has missed the confirmed slot. Task is now overdue. CSP needs to either push the executor, change the executor, or take it over themselves.

## Banner
| Field | Value |
|---|---|
| Icon | `Warning` |
| Title | `Rajesh ने कनेक्शन सेटअप में देरी कर दी` |
| Subtitle | `सेटअप को जल्दी पूरा करवाएं` |
| Accent | `stateNegative` |
| Bg | `bgNegative` |

Title is **event-style** (named person + past action verb), not status-style. `maxLines = 2` on title to fit long executor names without truncation.

## Why not "change executor" as primary framing
User pushback: "in drilldown for ins 1045, why is the delay resolution a team member change? Rajesh ne connection setup mei deri kar di should be used. Dusra technician chunein ya khud poora karein." → Surface the *event* first; offer the resolution path second.

## Executor row
- Executor section visible
- Status chip reads `कन्फर्म है` (the executor was confirmed — the problem is execution, not assignment)
- Inline call icon on **right edge of executor row**, in-line with the chip above it for visual consistency (user direction)
- Call icon uses 20dp `textSecondary` inside a 48dp tap target

## CTAs

| Slot | Text | Type | Action |
|---|---|---|---|
| Primary (sticky) | `टेक्निशियन बदलें` | Secondary | Opens ExecutorAssignmentSheet (routing bug fixed this session — previously opened slot sheet) |
| Overflow (⋮) | `नहीं कर पाएँगे` | Tertiary | Exit |

**Not** Destructive — changing executor is reversible.

## Routing bug (fixed this session)
`TaskDrilldownScreen.onPrimaryCtaClick` `when(state)` block had no `DELEGATED_OVERDUE` branch, so tapping "टेक्निशियन बदलें" fell through to the slot sheet. Fixed by adding:
```kotlin
"SLOT_CONFIRMED", "DELEGATED_OVERDUE" -> showExecutorSheet = true
```
